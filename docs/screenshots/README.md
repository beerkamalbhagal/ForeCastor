# Screenshot capture guide

The README references five screenshots by filename. Drop matching PNG
captures into this folder and GitHub will render them automatically.

| File                              | What to capture                                                                                       |
| --------------------------------- | ----------------------------------------------------------------------------------------------------- |
| `01-home-empty.png`               | First-run home dashboard: floating home chip in the top-left, search bar, title + subtitle, the empty hint "Search above to see cities you check appear here.", footer at the bottom. |
| `02-home-with-cities.png`         | Home dashboard with favorites + recents populated: the Favorites row (gold-tinted cards with the filled star) and the Recent row (each card with the X delete button and the outline star). |
| `03-search-autocomplete.png`      | Mid-search: query "Lon" with the dropdown showing matching cities (London, GB; London, CA; etc.) and one highlighted. |
| `04-weather-card.png`             | Full weather card: city + region + current date, large temperature with day/night pill + "feels like", sunrise/sunset pill, 2×2 conditions grid (humidity, wind, pressure, visibility), 10-day horizontal forecast strip. |
| `05-not-found.png`                | Not-found error screen: the title "Not Found", subtitle "The city you are looking for cannot be found.", and the app footer at the bottom. |

## Capture tips

- Run the app with `npm install && npm start` and open
  <http://localhost:3000> in Chrome.
- Viewport: 1280×800 (or 1440×900) desktop captures read best at
  GitHub's README @2x density.
- Use Chrome DevTools → "Capture full size screenshot" so the card
  isn't cropped to the viewport.
- For autocomplete (`03-`), make sure the dev server has been hit
  at least once so a valid OpenWeatherMap response fills the
  dropdown — otherwise the dropdown stays empty.

If you'd rather not capture them yourself, the same five files can
be regenerated by running the app and taking screenshots manually;
the `01-` to `05-` numbering is the only ordering constraint since
that's how the README references them.
